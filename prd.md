## 1. 개요 (Overview)

### 1.1 제품 정의
OOJOO FARM은 가정에서 식용작물(채소, 과채, 허브 등)을 재배하는 사용자를 위한 안드로이드 앱 **제품군**이다. 제품은 **두 종류의 기기에 각각 설치되는 두 개의 앱**으로 구성된다.

| 앱 | 설치 기기 | 역할 |
|----|-----------|------|
| **Master 앱** | 사용자 휴대폰 (갤럭시 S 등) | 사용자가 들고 다니며 식물을 모니터링·원격 지시·알림 수신·커뮤니티·마켓 이용. 슬레이브를 통해 식물을 재배하도록 지시 |
| **Slave 앱 (Farmer)** | 식물 앞에 고정된 안드로이드 기기 | 식물 앞에서 **지속적 모니터링** 및 **온디바이스 소형 AI로 스스로 관리**(관수·해충 퇴치·수확 감지). 하드웨어를 직접 제어 |

두 앱은 **무작위 생성 페어링 코드 공유**로 연결된다. 마스터 앱은 슬레이브 앱(Farmer)을 통해 식물을 재배하도록 하며, 슬레이브가 현장에서 자율 판단·실행하고 마스터는 그 과정을 감시·지시·기록한다.

### 1.2 아키텍처 개념도

```
┌─────────────────────────┐         ┌──────────────────────────────┐
│      Master 앱           │         │     Slave 앱 (Farmer)         │
│  (사용자 휴대폰)          │  페어링  │  (식물 앞 고정 기기)            │
│                         │  코드로   │                              │
│  · 대시보드/원격 제어     │ ◀──────▶ │  · 지속 카메라 모니터링        │
│  · 알림 수신(수확/해충)   │  동기화  │  · 온디바이스 소형 AI 자율판단  │
│  · 커뮤니티/마켓          │         │  · 자동 관수/해충 퇴치/수확감지 │
│  · 슬레이브 페어링/관리   │         │  · 하드웨어 직접 제어          │
└─────────────────────────┘         └──────────────┬───────────────┘
        ▲                                            │
        │ 푸시/상태                                    │ BLE/Wi-Fi
        │                                            ▼
   ┌────┴────┐                            ┌────────────────────┐
   │  클라우드 │  ← 날씨 API, 계정, 커뮤니티 │ 외부 하드웨어        │
   │  백엔드   │                            │ (급수밸브/Fan/Laser) │
   └─────────┘                            └────────────────────┘
```

### 1.3 배경 및 목적
- 가정용 식용작물 재배 인구가 증가하고 있으나, 물 공급·환경 관리·병해충 대응 등의 일관된 관리가 어려움
- 사용자 휴대폰을 식물 앞에 계속 고정해 두는 것은 비현실적 → **현장 상주용 슬레이브 기기**와 **휴대용 마스터 기기**를 분리
- 슬레이브는 클라우드/마스터 통신이 끊겨도 **온디바이스 소형 AI로 자율 관리** 가능(신뢰성 확보)
- 별도 고가 IoT 전용기기 없이 중고/저가 안드로이드폰을 슬레이브로 재활용해 진입 장벽을 낮춤
- 지역 기반 커뮤니티와 용품 마켓으로 사용자 체류와 수익을 창출

### 1.4 타겟 플랫폼
| 구분 | OS | 기기 |
|------|----|------|
| Master 앱 | Android minSdk 26 (8.0+) | Samsung Galaxy S 시리즈 우선, 일반 안드로이드폰 |
| Slave(Farmer) 앱 | Android minSdk 24 (7.0+) | 고정 전용. 중고/저가폰, 화면 크기 무관(헤드리스 동작 지원). 충전 상시 연결 권장 |

### 1.5 비전(Vision)
"마스터 폰은 지시하고, Farmer 폰은 현장에서 스스로 키운다. 누구나 집에서 식용작물을 심고, 키우고, 나누고, 거둔다."

1. 슬레이브(Farmer)가 온디바이스 AI로 식물 상태를 지속 인식하고 자율 관수

## 4. 핵심 기능 정의 (Core Features)

> 기능은 **Master 앱 영역(M)**과 **Slave(Farmer) 앱 영역(S)**, 그리고 **공통/연결(C)**로 구분한다.

### 4.1 [C] 마스터-슬레이브 페어링 (무작위 코드 공유)
**설명**: 마스터 앱과 슬레이브 앱을 안전하게 연결한다. 마스터가 무작위 생성 코드를 발급/표시하고, 슬레이브가 코드를 입력(또는 QR 스캔)해 연결한다.

**기능 상세**
- 마스터: "Farmer 추가" → 6~8자리 무작위 페어링 코드(또는 QR) 생성, 유효시간 10분
- 슬레이브: 코드 입력 화면 → 코드 입력 → 백엔드 인증 → 세션 키 발급
- 연결 후 기기 식별(별명 부여, 예: "베란대 토마토 Farmer")
- 1:N 지원: 마스터 1대에 여러 슬레이브 연결 가능
- 연결 끊김/재연결 처리, 기기 교체 시 이관
- 보안: 코드 1회용, 세션 키 암호화, 비정상 접속 차단

### 4.2 [S] 지속 카메라 모니터링
**설명**: 슬레이브가 식물 앞에서 후방 카메라로 지속 캡처·분석한다.

**기능 상세**
- 캡처 주기: 기본 1시간(설정 10분~6시간), 해충 감지용 짧은 주기(기본 15분) 별도 운영
- 화면 항상 켜짐/헤드리스 동작 모드 토글
- 카메라 고정/프레임 보정 가이드
- 분석 결과(상태/열매/곤충)를 로컬 저장 후 마스터로 전파
- 식물 상태 판별 항목: 수분 스트레스, 잎 색상 이상, 성장 단계

### 4.3 [S] 온디바이스 소형 AI 자율 판단 & 관리
**설명**: 슬레이브에 탑재된 **작은 AI**가 클라우드 없이 현장에서 스스로 판단해 관수·퇴치·수확 감지를 실행한다. 이것이 슬레이브가 "스스로 관리"할 수 있는 핵심.

**자율 판단 범위**
- 수분 부족 판정 → 자동 관수 실행(공급량은 로컬 날씨 캐시/온습도 보정)
- 곤충 탐지 → Fan/Laser 자동 대응 (사용자가 승인 모드인 경우)
- 열매 수확 적기 도달 → 마스터로 알림 (실행은 사람)
- 이상 징후(병해/심각 시듦) → 마스터 긴급 알림

**자율 실행 정책(사용자 설정)**
| 항목 | 자동 / 확인 | 비고 |
|------|-------------|------|
| 관수 | 자동(기본) | 마스터 오버라이드 가능, Fail-safe 밸브 타이머 |
| Fan 퇴치 | 자동(기본) | 안전 범위 내 |
| Laser 퇴치 | 확인 필요(기본) | 마스터 승인 후 실행, 안전 규격 준수 |
| 수확 알림 | 자동 알림 | 실행은 사용자 |
| 긴급 이상 | 알림만 | 사람 대응 |

**AI 모델 (경량·온디바이스)**
- 식물 상태 분류(수분/이상) — MobileNet 계열, 수 MB
- 열매 탐지 — 경량 객체탐지(NanoDet/MobileDet)
- 곤충 탐지 — 경량 객체탐지(해충/익충)
- 레이저 타겟 좌표 변환 — 이미지 좌표 → 물리 좌표
- 정기적 모델 업데이트(마스터/클라우드 경유 배포)
- 사용자 라벨링 피드백으로 개선 데이터 확보

### 4.4 [S+M] 날씨 연동 관수량 조절
**설명**: 날씨 API(온도·습도·강수)로 관수량·시점을 조절. 날씨 정보는 마스터/클라우드에서 슬레이브로 주기적 전파되며, 슬레이브는 오프라인일 때 마지막 캐시값으로 동작한다.

**관수량 알고리즘**: `기준량 × 온도가중치 × 습도가중치 × 강수보정`
- 고온(28℃↑): ×1.2~1.5 / 저온(15℃↓): ×0.6~0.8
- 습도 70%↑: ×0.7 / 강수 예정: 연기 또는 ×0.3
- 실내 재배 시 사용자 온습도 수동 보정
- 관수 로그 + 날씨 조건을 함께 기록해 마스터에서 피드백 제공

### 4.5 [S] 외부 하드웨어 직접 제어
**설명**: 슬레이브가 현장에서 외부 하드웨어를 **직접** 제어한다(마스터는 원격 지시만). 마스터→슬레이브→하드웨어 체인.

**제어 대상**
| 장치 | 동작 | 안전 |
|------|------|------|
| 급수 솔레노이드 밸브/펌프 | On/Off, 공급 시간 | 과수 방지 타이머(Fail-safe) |
| 외부 Fan | On/Off, 풍량 | 과열 보호 |
| 외부 Laser | On/Off, 타겟 좌표 | 출력·시간 제한, 사용자 동의 |

**통신**: BLE(ESP32) 또는 Wi-Fi 릴레이. 슬레이브가 로컬에서 즉시 제어.
**페어링/상태**: 슬레이브가 하드웨어 페어링 관리, 연결 상태를 마스터에 보고.

### 4.6 [M] 원격 모니터링 대시보드 & 제어
**설명**: 마스터 앱은 슬레이브가 수집한 상태/이벤트를 받아 대시보드로 보여주고, 원격 지시(관수/퇴치/모드 변경)를 내린다. (현재 UX/UI 프로토타입이 이 영역에 해당)

**기능 상세**
- 식물/기기 요약, 날씨, 빠른 관수(원격 지시)
- 식물 상세: 상태 지표, 카메라 분석 결과 뷰, 원격 관수/수동 제어
- 알림 센터: 수확/해충/관수 완료/날씨
- 슬레이브 상태: 온라인 여부, 마지막 캡처 시간, 하드웨어 연결, 배터리/충전
- 원격 지시 큐: 오프라인 슬레이브에 대한 지시 보관 후 재연결 시 실행

### 4.7 [M] 수확 시기 감지 & 알림
**설명**: 슬레이브가 열매를 추적하고 수확 적기 판정 시 마스터에 푸시 알림.

- 열매 탐지(수·크기·색상), 작물별 수확 적기 학습 데이터
- 마스터 푸시(FCM) + 인앱 알림 센터 + (선택) 워치
- 수확 기록 저장(수량/날짜/사진), 커뮤니티 자랑 연계

### 4.8 [M+S] 해충 감지 & 퇴치 제어
**설명**: 슬레이브가 곤충을 탐지해 Fan/Laser로 퇴치(자율 또는 마스터 승인). 마스터는 퇴치 이벤트를 수신·기록.

- 곤충 탐지(해충/익충 분류)
- Fan 모드: 자동 10초 가동(설정 가능)
- Laser 모드: 좌표 타겟팅, 마스터 승인 후 실행, 안전 규격 준수
- 탐지/퇴치 이벤트 로그(사진·시간·방식·결과) → 마스터 동기화

### 4.9 [M] 계정 & 지역 기반 커뮤니티
**설명**: 계정 기반 동작(마스터). 재배 지역 선택 후 주변 사용자와 작물 나눔.

- 회원가입/로그인(이메일·소셜)
- 프로필: 재배 지역(시/구), 관심 작물, 재배 이력
- 비로그인 모드(로컬 전용) 허용하되 커뮤니티·마켓은 계정 필요
- 나눔 마켓 / 작물 자랑 / 지역 피드(반경 설정 기본 5km)
- 신고/차단/평판 시스템

### 4.10 [M] 용품 마켓 & 제휴 링크 (수익 모델)
**설명**: 재배 용품 마켓과 제휴 링크로 수익 창출(마스터 앱).

**취급 품목**: 비료·배양토·씨앗/모종·물공급 탱크·호스·솔레노이드 밸브·펌프·팬·레이저 모듈·ESP32·조명·화분/틀·계측기, 그리고 **슬레이브용 중고/저가 안드로이드폰 추천**

**구매/연계**: 자체 상품 결제, 외부 제휴 링크(CPS/CPA), 재배 키트 번들 큐레이션, 재배 중 작물 맞춤 추천

**수익 채널**: 자체 마켓 마진, 제휴 수수료, (향후) 프리미엄 구독(무제한 Farmer 등록·상세 리포트)

## 5. 사용자 시나리오 (User Flows)

### 5.1 최초 설정 (마스터 + 슬레이브 페어링)
1. [M] 마스터 앱 설치 → 회원가입(또는 비로그인) → 재배 지역 선택
2. [S] 슬레이브 기기를 식물 앞에 고정, 충전 연결, Farmer 앱 설치
3. [S] Farmer 앱 실행 → 카메라 위치 가이드로 거치
4. [M] "Farmer 추가" → 무작위 페어링 코드(또는 QR) 생성
5. [S] 코드 입력 → 페어링 완료 → 기기 별명 부여
6. [M] 식물 등록(작물·식재일), 슬레이브에 식물 프로필 전파
7. [S] 외부 하드웨어(밸브/Fan/Laser) 페어링
8. [M] 자율 실행 정책(관수 자동/Laser 승인 등) 설정

### 5.2 일상 자율 관수
1. [S] 정기 캡처 → 온디바이스 AI 상태 분석
2. [S] 날씨 캐시로 관수량 산출 → 수분 부족 판정 → 밸브 On
3. [S] 공급 완료 → 밸브 Off(Fail-safe 타이머) → 로컬 기록
4. [S→M] 상태/관수 이벤트 전파 → 마스터 대시보드/히스토리 갱신
5. [M] 필요 시 원격 오버라이드(관수 취소/추가)

### 5.3 수확
1. [S] 열매 탐지/추적 지속 → 수확 적기 판정
2. [S→M] 마스터에 푸시 알림
3. [M] 알림 확인 → 수확 지시/기록 → 커뮤니티 자랑(선택)

### 5.4 해충 대응
1. [S] 곤충 탐지
2. [S] 정책에 따라 Fan 자동 실행 / Laser는 마스터 승인 요청
3. [S→M] 퇴치 이벤트 전파 → 마스터 기록
4. [M] 필요 시 수동 퇴치 지시

### 5.5 오프라인 자율 유지
1. 네트워크 단절 → 슬레이브는 온디바이스 AI+로컬 캐시로 자율 관리 지속
2. 재연결 시 마스터에 누적 이벤트 일괄 동기화

### 5.6 나눔 (마스터)
1. [M] "나눔합니다" 게시(작물·수량·위치)
2. 주변 사용자 댓글/쪽지 → 수령 후 완료 → 평판 반영

## 6. 기술 요구사항 (Technical Requirements)

### 6.1 앱 스택 (제안, 구현 시 확정)

| 구분 | Master 앱 | Slave(Farmer) 앱 |
|------|-----------|------------------|
| 언어 | Kotlin | Kotlin |
| UI | Jetpack Compose | Compose(단순) + 헤드리스 서비스 |
| 아키텍처 | MVVM + Clean | Foreground Service 중심, 경량 |
| 비전 | —(원격 뷰만) | **TFLite/ONNX 온디바이스 소형 AI** |
| 로컬 DB | Room | Room + 이벤트 큐 |
| 인증 | Firebase Auth | 세션 키(페어링) |
| 푸시 | FCM 수신 | FCM/또는 마스터→슬레이브 실시간 채널 |
| 하드웨어 통신 | (원격 지시만) | BLE(ESP32)/Wi-Fi 직접 |
| 카메라 | — | CameraX 지속 캡처 |

### 6.2 마스터-슬레이브 통신
- 페어링: 무작위 코드 → 백엔드 인증 → 세션 키 발급
- 실시간 상태/이벤트: Firebase Realtime DB / Firestore 또는 자체 WebSocket
- 원격 지시 큐: 오프라인 슬레이브용 보관(QoS, 재전송)
- 푸시: FCM(수확/해충/긴급)
- 보안: 세션 키 암호화, 기기 인증, 비정상 접속 차단

### 6.3 비전/AI 모델 (슬레이브 온디바이스)
| 작업 | 모델 유형 | 산출 | 비고 |
|------|-----------|------|------|
| 식물 상태 분류 | 경량 이미지 분류 | 수분스트레스/정상/이상 | MobileNet 계열 |
| 잎 질환/색상 | 분류/분할 | 질환 종류, 색상 지표 | |
| 열매 탐지 | 경량 객체탐지 | 박스·클래스·신뢰도 | NanoDet/MobileDet |
| 곤충 탐지 | 경량 객체탐지 | 박스·해충/익충 | |
| 레이저 타겟팅 | 좌표 변환 | 이미지→물리 좌표 | |
- 온디바이스 우선(지연·프라이버시·오프라인), 정기적 모델 업데이트(마스터/클라우드 경유)
- 총 모델 크기 목표 ≤ 30MB(저가폰 가동성 확보)

### 6.4 외부 하드웨어 규격 (슬레이브 연결)
- 컨트롤러: ESP32 (BLE + Wi-Fi)
- 급수: 12V 솔레노이드 밸브 + 릴레이 + 펌프
- 팬: 5V/12V DC 팬 + 릴레이
- 레이저: 저출력 모듈(안전 규격 Class 1 수준), 서보 조준
- 전원: 어댑터/배터리, 과류/과열 보호 회로

### 6.5 백엔드 (마스터/커뮤니티/마켓/페어링)
- 회원, 페어링 인증, 세션 관리
- 기기/식물/이벤트 동기화 스토어
- 커뮤니티(게시물·채팅·평판), 지역 검색(Geo)
- 마켓(결제·주문·제휴 링크 트래킹), CDN 이미지
- 날씨 API 연동 및 슬레이브 전파

### 6.6 퍼포먼스/신뢰성
- 슬레이브 비전 분석 1회 ≤ 5초(온디바이스)
- 슬레이브 BLE 제어 응답 ≤ 1초
- 마스터→슬레이브 원격 지시 지연 ≤ 2초(온라인 시)
- **슬레이브 오프라인 자율 유지 ≥ 24시간** (로컬 AI + 캐시)
- 관수 밸브 Fail-safe: 통신/전원 이상 시 자동 Off 타이머
- 슬레이브 충전 상시 권장, 배터리 임계 시 마스터 알림

### 6.7 보안/프라이버시
- 카메라 이미지는 슬레이브 온디바이스 처리 원칙, 클라우드 업로드 시 사용자 동의
- 위치 정보는 지역 단위 축약 표시(정밀 좌표 비공개)
- 페어링 코드 1회용·유효시간 제한, 세션 키 암호화
- 결제/계정 데이터 암호화, 개인정보 보호 규정 준수

## 7. UI/UX 가이드라인

### 7.1 Master 앱 구조 (현재 프로토타입 반영)
- **홈**: 식물/Farmer 요약, 날씨, 빠른 관수(원격 지시), 알림 카드
- **식물 관리**: 목록, 상세(상태/카메라 분석 뷰/원격 제어/히스토리)
- **Farmer 관리**: 기기 목록, 페어링, 상태(온라인/캡처/하드웨어/배터리), 자율 정책 설정
- **커뮤니티**: 지역 피드, 나눔, 자랑
- **마켓**: 추천, 카테고리, 상품/링크, 결제
- **설정**: 계정, Farmer, 하드웨어, 알림 규칙

### 7.2 Slave(Farmer) 앱 구조
- **최소 UI**: 페어링 코드 입력, 카메라 미리보기/고정 가이드, 하드웨어 페어링, 상태 표시(온디바이스 AI 동작/마지막 관수/이벤트)
- **헤드리스 모드**: 화면 끄고 Foreground Service로 지속 동작(충전 상시)
- **Kiosk/단순 모드**: 오류 시에도 재시작, 자동 부팅 후 앱 자동 실행 옵션

### 7.3 디자인 원칙
- 마스터: 직관적 식물 중심 대시보드, 초보자용 가이드/오버레이, 안전(레이저/관수) 확인 다이얼로그, 다크/라이트
- 슬레이브: 극단적 단순화, 설정 후에는 무인 동작, 오류 가시성
- 공통: 그린 컬러 테마, 라운드 카드, 부드러운 그림자

## 9. 릴리즈 로드맵 (Roadmap)

- [S] 급수 밸브 BLE 직접 제어 + Fail-safe
- 날씨 연동 관수량 추천(마스터→슬레이브 전파)

### Phase 2 — 자율 자동화/해충 (3개월)
- [S] 자율 관수 파이프라인(온디바이스 판정)

- [S] 곤충 탐지 + Fan/Laser 제어(자율/승인)
- [M] 히스토리/리포트, 오프라인 동기화

### Phase 3 — 커뮤니티 (2개월)
- [M] 지역 프로필, 나눔/자랑 피드, 채팅/신고/평판

### Phase 4 — 마켓/수익화 (2개월)
- [M] 자체 마켓 결제, 제휴 링크, 맞춤 추천, 슬레이브용 폰 번들

### Phase 5 — 고도화 (지속)
- AI 모델 정확도 개선, 다 Farmer 관리, 구독 플랜, 다작물 확장
